No source files are modified ‚Äî this is purely test infrastructure. The E2E suite is not a required CI gate; it runs only on `workflow_dispatch`.

## Implementation Notes

- `cy.seedMapDraft` uses `cy.window()` which is called post-visit; for the offline-with-draft
  test the draft must be set before `onMounted` runs, so `cy.visit(url, { onBeforeLoad })` is
  used directly instead of the custom command.
- Pull discard confirm test sets `unsaved = true` via Vue 3 dev-mode
  `__vueParentComponent.setupState.unsaved.value` ‚Äî only works against the Vite dev server.
- ESLint config extended with a Cypress-specific block (globals + `caughtErrorsIgnorePattern`)
  to cover `cypress/**/*.js` and `cypress.config.js`.
- `backups.cy.js` is a stretch test that requires the live dev server; all other tests use
  `cy.intercept` stubs so they run without a real server.
