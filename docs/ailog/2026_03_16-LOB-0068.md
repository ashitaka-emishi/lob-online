## Implementation Notes

The design workflow follows the same structural pattern as issue-intake (gather → draft/refine loop → gate → commit action) but produces a design doc on a `design/{slug}` branch rather than a GitHub issue. The skill is unowned (no agent parent) because it orchestrates the full lifecycle including git operations and PR creation — similar to `/plan-wrap` and `/pr-create`. Component type inference (`orchestrator` / `skill` / `agent`) is text classification based on the engineer's description; the skill proposes and the engineer confirms, keeping the human in the loop for the one decision that shapes the entire design.
