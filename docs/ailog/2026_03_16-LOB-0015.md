## AI Plan

This issue extends `scenario.json` with seven new fields grounded in LoB v2.0 and SM rules
(lighting schedule, night visibility cap, Fluke Stoppage grace period, initiative system,
Loose Cannon flag, loss recovery, and random events), extends the Zod schema to validate them,
adds a new `GET /api/tools/scenario-editor/data` + `PUT /api/tools/scenario-editor/data` server
route (guarded by `MAP_EDITOR_ENABLED`, versioned backups, `_savedAt` injection ‚Äî identical
pattern to the map editor), and creates `ScenarioEditorView.vue` with three panels: Turn
Structure, Lighting Schedule (add/edit/delete table), and Rules. The client uses the same
push/pull sync pattern as the map editor: `_savedAt` staleness check before push, ConfirmDialog
on overwrite/discard, offline banner on fetch failure, localStorage draft fallback. Tests cover
schema validation (valid/invalid/backward-compat), route (GET, PUT valid/invalid, backup
trimming), and view (offline, pull confirm, lighting CRUD, push staleness). Documentation is
resynced: README, CLAUDE.md, HLD ¬ß10 promoted from stub to spec, and a new
`docs/scenario-editor-design.md` is created.
