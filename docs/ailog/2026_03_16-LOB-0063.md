## Implementation Notes

Original 5 AC items completed as planned (export button removed, active tool name in header, accordion header fixed to "Hex Edit", selectedHex default for uncharted hexes, hex ID label color/size/visibility/even-column row offset). HCP 2a visual review surfaced 6 additional fixes that were folded into this branch:

1. **Legacy hexsides panel removed** from HexEditPanel — `hexsides` field still preserved in emitUpdate for data compatibility but the editing UI is gone.
2. **Seed hex as checkbox** — replaced the `★ Mark as Seed Hex` button with a standard checkbox under VP Hex / Entry Hex. CSS for `.seed-toggle`, `.seed-btn` and related removed.
3. **Seed hex color** — `seedHexIds` prop added to HexMapOverlay; seed hexes render with purple `#cc44ee` border. State managed in MapEditorView via a `Set<string>` with reactivity-safe replacement.
4. **Double-push confirmation bug** — `save()` condition changed from `serverSavedAt > localDraftSavedAt` to `unsaved.value && serverSavedAt > localDraftSavedAt`, so the "Server data is newer" dialog only appears when there are actual unsaved changes.
5. **North Offset in calibration** — `northOffset` (0–5) added to `DEFAULT_CALIBRATION` and CalibrationControls. WedgeEditor accepts a `northOffset` prop that rotates direction labels by k × 60° so they match the physical map orientation.
6. **Elevation label** — moved to bottom-center of hex (using corners[1]/corners[2] midpoint), larger font (11), dark green `#1a6b2a` with white halo (`paint-order: stroke`) for readability over varied terrain.
