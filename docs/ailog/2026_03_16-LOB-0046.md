## Implementation Notes

The skill is pure Markdown+bash — no JS extraction was needed. The key design decisions:

1. **Direct editing, not reporting** — per the refined scope, the skill edits docs in place rather than producing a findings table. This removes the friction of a separate "act on the report" step.
2. **Two separate skills** — `/doc-sync` handles `CLAUDE.md`, HLD, and design docs (diff-driven); `/ecosystem-docs-generate` handles `docs/claude-ecosystem/` (full rebuild). Both wired into `issue-implement` as `continue_with_warning` steps so neither blocks the workflow on failure.
3. **Smoke test incorporated** — running `/doc-sync` logic on this branch found that `CLAUDE.md` referenced `issue-implement.md` and contained "three HCPs" (stale — now five). Corrected and committed as part of the implementation commit.
4. **Registry entry added** — `doc-sync` registered in `.claude/agents/registry.json` so the orchestration runtime can dispatch it.
