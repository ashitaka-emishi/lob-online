## Implementation Notes

- `listBackups` task in `cypress.config.js` extended to accept a `prefix` parameter (default `'map-'`) so both map and scenario backup tests share one task without duplication.
- `vite-plugin-istanbul` is conditionally loaded only when `CYPRESS_COVERAGE=true` — the production build and normal `npm run dev` are unaffected.
- Server `/__coverage__` endpoint returns `global.__coverage__ || {}`. The global is populated by `@istanbuljs/esm-loader-hook` when the server is started with `NODE_OPTIONS='--experimental-loader=@istanbuljs/esm-loader-hook'` in CI. Local `test:e2e:coverage` runs collect client coverage only unless the server is started manually with that loader.
- `eslint.config.js` needed two additions: `global` in server globals (for `global.__coverage__`), and a new block for `*.config.js` / `client/*.config.js` files to add `process` (previously only `server/src/**` had `process`).
- `scripts/check-e2e-coverage.js` reads `coverage/coverage-summary.json` (generated by `@cypress/code-coverage` via nyc's `json-summary` reporter). It normalises path separators before matching so it works on Windows too.
- The `confirm-ok-btn` selector referenced in `backups.cy.js` handles the optional overwrite dialog by checking if the overlay exists in the DOM first.
