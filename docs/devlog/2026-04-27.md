## 12:20 — PR: Debt Final Closeout — 15 items, score 28 → 7

This PR closes every actionable open tech-debt item in the register, dropping the net open score from 28 to 7. The five items intentionally left open (#294, #295 performance hot-paths, and #205, #204, #201 OOB scale concerns) are pure performance/scale awareness flags with no correctness risk and no M4 blockers. Everything else that could be fixed without speculative engineering is now closed.

The work was organized into four phases matching the debt clusters. **Phase 1** targeted the map-test architecture items introduced with PR #297: shared panel CSS was extracted into a single `panel-shared.css` file used by all five SFCs via `@import`; a shared `mockFetch.js` test utility replaced five identical copy-pasted helpers; the `v-if`/`v-else-if` panel dispatch in `MapTestView.vue` was replaced with a `PANELS` registry and `<component :is>`; and the module-level `loadMap()`/`loadScenario()` calls in `mapTest.js` were moved into an Express startup guard that returns a 500 on any data-load failure rather than crashing the server at import time.

**Phase 2** addressed four engine quality items. The `ammoTypeShift` threshold check in `combat.js` was confirmed tautological by the domain-expert agent (the threshold is derived from the SP tier the firer already belongs to, making `firerSPs >= threshold` unconditionally true) and removed. `FORMATION_EFFECTS` and `ACTIVITY_EFFECTS` were extracted from `weapons.js` into a new `engine/tables/formations.js` module — they belong with formation rules, not weapon characteristics. Call-once startup guards (Set-based warn-on-repeat) were added to `loadMap()` and `loadScenario()` to surface per-request calls before M4 async refactoring lands. `hexEntryCost` gained a `dirIndex` 0–5 range validation that throws `RangeError` on bad input.

**Phase 3** cleaned up four Vue idiom issues in the table-test tool. A `pickMods()` helper was introduced in `tableTest.js` to allowlist-filter modifier objects before they reach the engine, replacing the previous `typeof x === 'object'` passthrough. `MoralePanel.vue` gained a `defaultModifiers()` factory so the `ref()` initializer and `reset()` share one source of truth instead of duplicating the ten-field object literal. `activePanel` was converted from a plain function to a `computed()` in `TableTestView.vue`, and all 11 eager panel imports were replaced with `defineAsyncComponent(() => import(...))`. The async component change required adding `__isTeleport: false` and `flushPromises()` calls to the test file — Vitest's strict mock proxy throws on undeclared property access, and Vue/VTU probes `__isTeleport` on the resolved module namespace before accessing `.default`.

**Phase 4** hardened the test fixtures and succession schema. `succession.schema.js` gained `.max(64)` on `id` and `name` and `.max(32)` on `baseLeaderId`, matching the conventions used by every other schema in the project. In `OobTreeNode.test.js`, the `_variants` fixture object was extracted into a `makeVariantsNode()` factory shared across three tests. In `oobTreeTransform.test.js`, `OOB_WITH_WJ_BRIGADE` and `OOB_WITH_RENO_BRIGADE` were refactored to extend `makeOob()` rather than duplicating the full OOB shape — each now only declares the brigade it adds as a delta.
