## 13:55 — PR: Map Editor UI Refactor — Tool Panel Cleanup & Feature Improvements

This PR closes out conductor track `map-editor-ui-refactor_20260323`, which was scoped on 2026-03-23 to deliver six targeted improvements to the map editor tool panels: removal of click/paint mode toggles and bulk elevation ops, building-terrain coexistence in the data model, a road rendering overhaul (wider outlined lines, 2+ edge threshold, bridge glyph), and a stream tool addition (ford as edge type with glyph rendering). The track was created before the implementation but the actual code changes were completed incrementally across the `map-editor-tool-closeout_20260323` and earlier sessions. This PR closes the tracking gap by adding the four test cases that were deferred and marking the plan complete.

The core design decision worth noting is that building-terrain coexistence was implemented via the existing `hexFeature: { type: 'building' }` field rather than the originally planned `building: z.boolean()` migration. Both approaches achieve the same user-visible behavior — a hex can carry both a terrain type and a building overlay simultaneously — but the `hexFeature` path was already in place and validated by the schema. Migrating to a separate boolean would have required a data migration pass over `map.json` and offered no functional improvement at this stage.

The road rendering 2+ edge threshold (suppressing isolated single-edge roads) was implemented in `HexMapOverlay.vue` via `roadEdgeCountMap`, a computed that counts canonical edge ownership across neighbors. The threshold test added in this PR explicitly covers the edge case of exactly one road edge (not just zero), which the prior test suite left uncovered. Bridge and ford glyphs are rendered as SVG `<text>` elements in dedicated layer groups (`layer-bridge-glyphs`, `layer-ford-glyphs`), rotated to align perpendicular to their respective edge directions using pre-computed angle constants.

The four new tests added here are the only code changes in this PR. All production code was already in place. The tests were identified as gaps during the track closeout review: 1-edge road suppression, bridge glyph presence, ford glyph presence, and building-over-woods terrain preservation. Each test mounts the relevant component with a minimal fixture and asserts on rendered SVG structure or emitted values.

Deferred: no items. The full acceptance criteria for issues #180–#185 are now covered by tests and the CI gates pass cleanly.
