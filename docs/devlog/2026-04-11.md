# 2026-04-11

## 06:07 — PR: M3 engine modules — hex, movement, LOS, and game tables

This PR delivers the complete M3 rules engine foundation: ten pure-function server modules covering every mechanical layer needed to execute a turn of Line of Battle v2.0 — from hex geometry through movement costs and line-of-sight to all six game tables. The work was scoped as track `m3-engine_20260410` and executed TDD with a domain-expert consultation before each table module was coded. 1670 tests pass; engine coverage sits at 96% statements / 95% branches.

The two lowest-level modules, `hex.js` and `scenario.js`, establish the coordinate system and data foundation. `hex.js` implements flat-top EVEN_Q offset cube coordinates, six-direction neighbor lookup with boundary validation, hex-line interpolation (used by LOS), and a Dijkstra skeleton that the movement engine drives. The cube-coordinate math is subtle — game IDs are one-indexed col/row from the bottom-left, which requires a two-step conversion through a honeycomb-grid intermediate — and it was validated against known SM hex pairs before any higher module was written.

`movement.js` is the most complex module. It reads all costs from `scenario.json` (no hardcoded terrain numbers anywhere in engine code), implements the road/pike/trail `"ot"` sentinel logic, derives slope hexside types from `wedgeElevations` at runtime rather than storing them as explicit edge features, and handles the road slope penalty correctly — road movement suppresses normal slope hexside costs and substitutes a lighter per-slope-type penalty. Several bugs were found and fixed during TDD: the penalty loop initially fired on the road feature itself, and road + slope was giving incorrect totals before the suppression guard was added.

`los.js` corrects a design error present in the existing client `los.js`: terrain height bonuses (woods +1, orchard +1 per SM §1.4) apply **only to intermediate obstacle hexes**, never to the observer or target End Point heights. This was confirmed authoritatively by the domain-expert before implementation. The server module also implements the orchard first-hex ignore rule (SM §1.4), the Same Hill rule (LOB §4.2c, strictly-greater comparison), and adjacent-hex always-clear (LOB §4.2a). Vertical slopes are movement-only and do not block LOS — another ruling that deviates from naive intuition.

The six table modules in `tables/` were each preceded by a domain-expert consultation that verified exact cell values before a line of implementation was written. Highlights: the Morale Table extends to rolls 13–14 (only reachable via modifiers), the Additive Morale Effects Chart has an asterisk rule suppressing retreats/losses when a Blood Lust unit transitions to a worse state, and the Combat Table's lettered columns A–D are only reachable through right-shifts from the numeric columns. The `command.js` module encodes the SM override that all army commanders are rated Normal (LOB_GAME_UPDATES SM), making the Order Delivery table effectively single-valued for SM. No map test tool or table test tool UI was built in this PR — those are planned for a future track.
