## 01:58 — PR: Counter Auto-Detection Script Revision (#225)

This PR delivers a complete revision of `scripts/detect-counters.js` — the one-time vision
script that maps counter image files to OOB unit records via Claude's vision API. The
original script (from track counter-detect_20260325) had a 76-match rate on the first run
because it sent all 468 images against an undifferentiated list of 170 units, making it too
hard for the model to distinguish one blue infantry counter from another. The revision
attacks the problem at its root by pre-classifying every image before any API call is made.

Phase 1 added two pure-function helpers: `classifyImage(filename)` parses filename patterns
(C## → CSA front, U## → Union front, CS1-Front\_\* / CS1-Back\_\* → face from name) to
return an `{ army, face }` classification without any network call, and
`detectArmyByColor(client, imagePath)` issues a lightweight second Claude call only for the
ambiguous CS1-\* files where the filename gives no army information. This pre-classification
step cuts the candidate roster from 170 to either 75 CSA or 95 USA units, a 55–45% reduction
that materially improves prompt discrimination.

Phase 2 replaced the flat unit-list prompt with one that includes each unit's full command
chain (corps › division › brigade) so the model can use organizational context — "which
regiment in Hood's division?" is a far more tractable question than "which of 170 units is
this?". The `buildRoster` helper assembles the filtered, annotated candidate list. Phase 2
also fixed a latent JSON parse failure: Claude was responding with valid JSON followed by
explanation prose, which caused `JSON.parse()` to throw on ~293 images. The fix uses a
`/\{[^}]*\}/` regex to extract the first `{…}` object before parsing.

Phase 3 executed the revised script. The first pass matched 76 images; after the JSON fix
and a `--force` re-run, the final tally was 269 images matched out of 468 (CSA: 25 front +
27 back out of 75; USA: 31 front + 14 back out of 95; leaders: 8 of 48). The confidence
threshold was set to 0.0 to capture every best-guess match — the intent is to seed as many
counterRef values as possible for manual verification in the OOB Editor, not to filter for
certainty. The ~199 remaining null counterRefs are left for manual linkage. Twenty new unit
tests covering all helpers were added and vitest.config.js was updated to include
`scripts/**/*.test.js` in the server test project.
