# 2026-03-30

## 07:05 — PR: OOB Data Model Completion and Server Routes (#189 #191)

This PR delivers the server-side foundation the OOB editor has been waiting for: three new
Express routes (`/api/oob`, `/api/leaders`, `/api/counters`) and the schema + data changes that
make the full command hierarchy addressable by the editor. The OOB store already called
`/api/oob/data` and `/api/leaders/data` as its first-choice fetch — those routes just didn't
exist yet, leaving the editor permanently on its local JSON fallback. Now the round-trip works.

The schema work for issue #189 required two passes over `oob.json`. The first patch script
detected command nodes by checking for `brigades`, `divisions`, or `corpsUnits` keys — which
correctly caught corps, divisions, and the cavalry division, but missed the 32 brigade nodes
entirely (brigades have `regiments` + `wreckThreshold`, not the higher-level keys). A second
pass with an updated predicate caught all 45 command nodes. The integration test added to
`oob.schema.test.js` (which parses the real file and surfaces the first Zod error path on
failure) would have caught this even if the manual check hadn't — it's a useful pattern for
schema tests on non-trivial data files.

The three route files follow `mapEditor.js` exactly: 60 req/min rate limiter, max-20 backup
rotation with timestamp-based filenames, `_savedAt` epoch stamp on write, and a shared
`backups/` directory. The backup prefix differs per file (`oob-`, `leaders-`, `map-`) so a
single directory holds all three without collision. The counters route uses `multer` (newly
added) with disk storage, a `.jpg`/`.png`-only file filter, and a 500 KB size limit. File
type rejection works through `multer`'s `fileFilter` callback returning `false` rather than an
error, with the route checking `req.file` afterward — this keeps multer's error handling clean
and avoids swallowing size-limit errors.

Issue #190 (leaders `counterRef`) turned out to be already complete from PR #208 — all 48
leader records had `counterRef: null` in the data and the schema already defined `LeaderCounterRef`
with all eight confidence fields. Closing it was purely administrative. The remaining work before
the OOB editor is feature-complete is two UI tracks: the succession list component (#195) and the
leader detail panel (#194).
