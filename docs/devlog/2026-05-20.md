# 2026-05-20

## 22:45 — PR: Architecture debt sprint — issues #370 #322 #393 #394

This PR closes the four highest-value actionable debt items identified after the M5 engine
work, plus a process fix that had been sitting uncommitted in the working tree. Total: 10
debt points removed, net open score drops from 24 to 19 across 10 remaining items. All
remaining open items are either M6-blocked engine stubs or low-priority performance notes.

The most structurally significant change is the scenario cache extraction (#393). The lazy
`_scenario` cache and its `clearScenarioCache()` invalidator had been living inside
`routes/games.js` since PR #337 when it was first added as a quick fix. That placement
created a route-to-route import: `scenarioEditor.js` imported `clearScenarioCache` from
`games.js`, meaning a route module was depending on another route module purely for side
state management. Moving the cache to `engine/scenario.js` — where `loadScenario()` already
lives — gives the cache a natural home and makes the dependency direction clean (routes →
engine). One subtlety: `loadScenario()` has a call-count guard from PR #293 that emits a
`console.warn` if called more than once per path. The new `clearScenarioCache()` explicitly
deletes the path from the guard's Set before resetting the cache, so intentional reloads
after a scenario-editor save don't produce spurious warnings.

The `isOrderHolder()` extraction (#394) is smaller but matters for M6: once order-resolution
and activity-phase query helpers land, they'll want a natural place to import from. An `init`
module is a confusing home for a read-side query predicate. The new `engine/queries.js` gives
those helpers a correct address. The move was clean — one consumer in tests, one pointer
comment in the schema — and all behavioral tests were kept in `init.test.js` since they
exercise the predicate in the context of real `initGameState()` output. A minimal smoke test
was added to `queries.test.js` to confirm the new module exports correctly.

The `pickMods()` bounds fix (#322) is a dev-tool-only change in `routes/tableTest.js`. The
function already allowed-listed modifier keys and coerced them to numbers, but applied no
domain bounds, so a value of 99 for `leaderMoraleValue` (which should be 0–4 per LOB §6.1)
would reach the engine unclamped. The fix adds an optional `bounds` parameter and applies
`Math.max/min` clamping at the morale call site. Values are silently clamped rather than
rejected — the table-test tool is a debugging aid, not a user-facing endpoint, and clamping
is less disruptive than a 400 error when a developer fat-fingers a value.

The `complexDefense` comment fix (#370) was unblocked by reading the source PDF
(`lob-game-specific-updates.pdf`, SM section p.4): "All those 'Complex' defense orders should
be Move Orders." The comment now cites the exact text and page. The `.claude` workflow fix
(uncommitted changes that added the `tech-debt` label step to `tech-debt-report.md` and
`agentic-quality-rails.md`) was committed as the first change on the branch — it had been
waiting for a suitable PR to ride along with.
