# 2026-03-24

## 10:11 — PR: Map editor rendering and interaction polish

This PR delivers a second wave of map editor UX polish on top of the #180–#185 refactor merged
yesterday. The changes fall into four areas: road rendering correctness, glyph sizing and
orientation, terrain tool fixes, and interaction discipline.

The most significant fix is to the through-hex road rendering. The previous implementation only
iterated the three canonical edge directions (N, NE, SE) per hex when building center-to-midpoint
segments, meaning roads on a hex's S, SW, or NW edges were never drawn from that hex's center
outward. The fix extends `useEdgeLineLayer` to accept an optional `cellById` map and `gridSpec`,
then looks up mirror-edge features from adjacent hexes' canonical faces when `includeMirrors` is
true (through-hex mode only). This restores the full six-direction spoke for every routed hex.

Bridge and ford glyphs were resized from font-size 9 to 24 and given a rotation transform so
the `][` symbol aligns along the hex edge direction rather than always rendering horizontally. A
`glyphTransform` helper in `HexMapOverlay` computes `rotate(angle, cx, cy)` based on a fixed
map (N/S → 0°, NE/SW → 60°, SE/NW → −60°). Road line weights were also increased substantially
(trail 5→8 px, road 6→10 px, pike 7→12 px) so roads read clearly against the map image. The
building icon was scaled to match (font-size 10→24) and recolored to black-on-white to
distinguish it from the terrain fill beneath it.

Interaction discipline was tightened in two ways. First, `paintMode: ref('click')` is now
explicitly passed to `useHexInteraction` from `MapEditorView`, closing a bug where the composable
defaulted to `'paint'` mode and would apply terrain or elevation to every hex the cursor drifted
over. Second, hex selection is removed from elevation and paint click handlers — only the LOS
tool and wedge/select modes may set `selectedHexIds`. The LOS tool itself was also broken because
`'losTest'` was absent from `INTERACTIVE_PANELS`, preventing `HexMapOverlay` from emitting click
events when that panel was open; that is now corrected.

Two data issues were fixed: the orchard terrain color key was `orchards` (no match) while the
terrain type string is `orchard`, so orchard hexes showed no fill; the key is now corrected and
the color changed to a light green. Road groups were also removed from the base overlay config's
`ALL_EDGE_GROUPS` (renamed `BASE_EDGE_GROUPS`) so the old 20/80 along-edge road segments no
longer appear when no tool panel is active — roads are exclusively rendered via the road panel's
through-hex style.
