# 2026-04-10

## 01:11 — PR: M2 Closeout Debt Sprint

This PR closes all remaining M2 technical debt — twelve issues in total — clearing the register
before the project advances to M3 game-engine work. The motivation was to start the new milestone
with zero open debt so that future PR reviews don't carry forward noise from the previous sprint.
Five of the twelve issues (#221 #220 #241 #214 #216) were discovered during pre-implementation
audit to be already resolved by prior PRs (the editorRouteFactory refactor in #261 and store tests
in #252), and were closed without code changes. The remaining seven required new implementation.

Schema deduplication (#242 #258) was the first phase. `CommandLevel` and both `CounterRef` shapes
were extracted into a new `shared.schema.js` to eliminate the duplication between
`leaders.schema.js` and `succession.schema.js`. A key constraint here was that the two CounterRef
shapes are _not_ identical — leaders require all promoted-counter fields while succession makes them
optional — so two named exports (`LeaderCounterRef`, `SuccessionCounterRef`) were necessary rather
than a single shared shape. `specialRules` was tightened from `z.unknown()` to
`z.union([z.string(), z.boolean()])` after inspecting the actual data files to confirm those are the
only value types in use.

Atomic writes (#251) replaced the direct `writeFile(filePath, ...)` call in `editorRouteFactory.js`
with a write-to-`.tmp`-then-rename pattern. `fs.rename` is atomic on the same filesystem, so a
crash mid-write leaves the original file intact. The `.tmp` file is cleaned up on failure. All five
route-specific test files needed `rename: vi.fn()` added to their `fs/promises` mock blocks — this
was discovered only after running the full suite, since the factory is shared but the mocks are
per-file.

The client/server contract test (#257) required extracting `isValidSidedObjectShape` and
`isValidSuccessionShape` out of `useOobPersistence.js` into a new pure-JS `oobValidators.js` with
no Vue dependency. This made the functions importable from the server (Node) test environment,
enabling `oob-client-contract.test.js` to import both the client validators and the Zod schemas in
one file and assert that valid server-parsed objects pass client validation and structurally
broken objects fail both sides. The offline-detection feature (#207) added an `isOffline` ref to
`useOobPersistence` — set when the server is unreachable and localStorage is used as fallback,
cleared on successful server contact, with push blocked while offline.

The final cleanup items were test quality improvements: replacing ordinal-based `mockFetch` in
`useOobStore.test.js` with URL-keyed dispatch (#254) so tests no longer depend on the order of
fetch calls, and extracting a shared `MINIMAL_SUCCESSION`/`MINIMAL_VARIANT` fixture into
`succession.fixtures.js` (#253) to remove duplication between the schema test file and any future
consumers. No new debt was introduced; the register is now empty at M2 close.
