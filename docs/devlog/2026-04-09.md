# 2026-04-09

## 11:56 — PR: M2 Debt Sprint — Score-3 Items (#237 #245 #247)

This PR closes the three highest-priority debt items deferred from the M2 OOB editor phase, all of which the tech-debt report flagged as risks that should be resolved before game engine work begins. The three items span a bug, an architectural duplication, and a missing test file — each in a different layer of the stack, but all scoped tightly enough to land in a single sprint.

The `processUSACavDiv` bug (#237) was the most impactful fix. The function had inlined Pleasonton's variant-attachment logic — duplicating exactly what `withLeader` already does — and attached Farnsworth's leader without any variant lookup at all. Both are now delegated to `withLeader`, which means Farnsworth succession variants will render correctly in the OOB tree. The fix is about 15 lines shorter than the original and adds three new tests: Farnsworth gets `_leader` when present, Farnsworth gets `_leader._variants` when a succession variant exists, and a regression guard confirming Pleasonton variants still render after the refactor.

The `createEditorRoute` factory (#245) was the third time the same ~50-line backup-rotation PUT handler appeared verbatim across `oobEditor.js`, `leadersEditor.js`, and `successionEditor.js`. The factory takes `{ schema, filePath, filePrefix, backupDir, maxBackups }` and returns a configured Express Router. Each route file now consists of constants, a rate-limiter, and a single `createEditorRoute(...)` call. As a side-effect, this also fixes #241: the factory wraps the final `writeFile` in try/catch, which the individual route files all omitted. Twelve new factory tests cover the GET and PUT paths independently of any specific schema.

The `useOobPersistence.test.js` file (#247) fills a notable gap: the composable is the single component responsible for all data I/O in the OOB editor, yet it had no dedicated test file after the succession feature was added in PR #236. Twelve tests now cover the L1 server path (including non-OK succession endpoint), the L2 localStorage path (including invalid succession shape rejection), push skipping a null succession endpoint, `localStorage.removeItem` being called on successful push, and `pullFromServer` failure resetting `isSyncing`. The tests use a URL-keyed fetch mock rather than ordinal matching, which avoids the subtle failure mode where reordering fetches silently returns wrong fixture data.

With these three fixes merged, the M2 score-3 debt is retired. The remaining 11 open items (#241 was incidentally fixed by #245) are all score-1 or score-2 and can be deferred until after M3 game engine work begins or batched into a later cleanup sprint.
