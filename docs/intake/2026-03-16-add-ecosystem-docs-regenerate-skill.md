## Description

The `docs/claude-ecosystem/` folder was written by hand and can drift from the actual agent
definitions, skill files, and workflow JSON as the ecosystem evolves. A
`/ecosystem-docs-generate` skill would reverse-engineer the current state of the ecosystem
from its source-of-truth files — `docs/agents/*/design.md`, `.claude/agents/*.md`,
`.claude/agents/registry.json`, `.claude/commands/*.md`, and `docs/workflows/*/` — and
regenerate or update every file in `docs/claude-ecosystem/` to match. This closes the same
drift gap that `/agent-sync` and `/agent-regenerate` address for the agent layer, but at the
documentation layer. The existing `docs/claude-ecosystem/` folder serves as the authoritative
template for the generated output.

## Acceptance criteria

- A skill file exists at `.claude/commands/ecosystem-docs-generate.md`
- The skill is registered in `.claude/agents/registry.json` with `id: "ecosystem-docs-generate"`, `type: "skill"`
- Running `/ecosystem-docs-generate` reads the following source-of-truth inputs:
  - `docs/agents/*/design.md` — per-agent details (purpose, responsibilities, allowed tools)
  - `.claude/agents/registry.json` — canonical agent and skill index
  - `.claude/commands/*.md` — skill descriptions and step sequences
  - `docs/workflows/*/` — workflow JSON and states docs
- The skill regenerates all files in `docs/claude-ecosystem/` to reflect current state:
  - `README.md` — hub index table and quick-start command list
  - `agents.md` — one section per agent; purpose, collaborators, owned skills, allowed tools
  - `skills.md` — one row per skill; purpose, owning agent, Claude Code invocation, manual equivalent
  - `network-diagram.md` — Mermaid diagrams: agent ownership, skill dependencies, SDLC sequence
  - `guardrails-and-logging.md` — regenerated from HCP gates in workflow JSON and ailog conventions
  - `blog.md` and tutorial files — **not** regenerated (human-authored prose; left untouched)
- The skill explicitly marks `docs/` as the source of truth in its introductory comment and in
  the generated file headers (`> Auto-generated by /ecosystem-docs-generate — do not edit by hand`)
- Each generated file calls out the three-layer relationship: agents (scope) → skills
  (procedure) → orchestration (sequencing)
- If a target file already exists, the skill diffs the proposed content against the current
  file and reports what changed before writing, so the engineer can review the delta
- The skill is purely read/write over documentation files; it never modifies `.js`, `.vue`,
  `.json` data files, or `.claude/agents/*.md` agent definitions
- `/dev-build` passes after the skill is added (format, lint, build clean)
