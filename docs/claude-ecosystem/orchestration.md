The orchestration layer sequences agents and skills into pipelines with blocking gate
checkpoints where the engineer must provide an explicit decision before the pipeline
continues. **Agents** define what each subprocess is allowed to do; **skills** compose
agents into procedures; **orchestration** defines when and in what order to invoke them.

lob-online's orchestration layer is provided by the
[wshobson/agents](https://github.com/wshobson/agents) plugin marketplace. Two plugins are
installed: `conductor` (SDLC track management) and `agent-teams` (parallel multi-agent
workflows).

## `conductor` — SDLC Track Management

Conductor organises work into **tracks** — self-contained units of implementation with a
spec, a phased plan, and task-level TDD workflow. Tracks live in `conductor/tracks/`.

### Key commands

| Command                                 | Purpose                                                            |
| --------------------------------------- | ------------------------------------------------------------------ |
| `/conductor:new-track`                  | Scope a new feature: gather intent, write spec, create phased plan |
| `/conductor:implement`                  | Execute tasks from the current track's plan following TDD          |
| `/conductor:status`                     | Display project status, active tracks, and next actions            |
| `/conductor:manage`                     | Archive, restore, delete, rename, or clean up tracks               |
| `/conductor:revert`                     | Git-aware undo by logical work unit (track, phase, or task)        |
| `/conductor:setup`                      | Initialize project with conductor context artifacts                |
| `/conductor:context-driven-development` | Create/maintain `conductor/` context docs                          |

### Track lifecycle

1. **Spec** — `/conductor:new-track` gathers intent and writes `conductor/tracks/{slug}/spec.md`
2. **Plan** — conductor creates a phased `plan.md` with discrete tasks
3. **Implement** — `/conductor:implement` works through tasks; commits after each
4. **Checkpoint** — conductor pauses at phase boundaries for engineer approval
5. **Complete** — all tasks done; PR created, reviewed, merged

### Replacing the old workflow

| Old                    | New                                             |
| ---------------------- | ----------------------------------------------- |
| `/issue-intake`        | `/conductor:new-track` (spec creation step)     |
| `/issue-implement <n>` | `/conductor:new-track` → `/conductor:implement` |
| Hand-rolled HCPs (6)   | Conductor phase checkpoints                     |

See `docs/migration-wshobson-agents.md` for the full command mapping.

## `agent-teams` — Parallel Multi-Agent Workflows

Agent-teams spawns multiple specialised reviewer or implementer agents in parallel,
coordinates their work via file ownership boundaries, and synthesises results.

> **Enable:** set `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` in `.env` or your shell.

### Key commands

| Command          | Purpose                                                                           |
| ---------------- | --------------------------------------------------------------------------------- |
| `/team-review`   | Multi-dimension parallel PR review (security, performance, architecture, testing) |
| `/team-debug`    | Parallel hypothesis debugging: competing investigators, root cause arbitration    |
| `/team-feature`  | Parallel feature development with file ownership boundaries                       |
| `/team-spawn`    | Spawn a team using a preset (review, debug, feature, fullstack, etc.)             |
| `/team-status`   | Display team members, task status, and progress                                   |
| `/team-shutdown` | Gracefully shut down an agent team and collect results                            |
| `/team-delegate` | Task delegation dashboard for managing team workload                              |

### Replacing the old workflow

| Old                 | New                                                     |
| ------------------- | ------------------------------------------------------- |
| `/pr-review`        | `/team-review`                                          |
| `code-review` agent | `agent-teams:team-reviewer` (invoked by `/team-review`) |
| _(no equivalent)_   | `/team-debug`                                           |
| _(no equivalent)_   | `/team-feature`                                         |

## Plugin Installation

On a new machine:

```
/plugin marketplace add wshobson/agents
/plugin install conductor
/plugin install agent-teams
```

Set `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` in `.env` (or your shell) to enable
agent-teams parallel workflows.
