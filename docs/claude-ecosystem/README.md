lob-online uses a structured set of Claude Code **agents** and **skills** to automate its
software development lifecycle — from scoping an idea to merging a reviewed, tested pull
request. Agents are specialised AI subprocesses with focused responsibilities and explicit
tool allowlists; skills are reusable Markdown prompt files that encode step-by-step
procedures. The primary SDLC orchestration layer is provided by the
[wshobson/agents](https://github.com/wshobson/agents) plugin marketplace (`conductor` and
`agent-teams`). Lob-specific agents and skills handle the domain and devops work that
plugins cannot.

## How the SDLC Works

A feature moves through four phases. **Design:** for any new or changed component, the
engineer runs `/design`. The skill gathers the intent, infers the component type, creates
`docs/designs/{slug}.md` from the canonical template, iterates until the engineer approves,
then commits on `design/{slug}` and opens a PR. **Track creation:** the engineer invokes
`/conductor:new-track` to scope the work into a tracked implementation plan with a spec and
phased tasks. The `rules-lawyer` agent is consulted automatically if the feature touches
game mechanics. **Implementation:** `/conductor:implement` executes tasks within the track,
following TDD workflow. After implementation, `/doc-sync` and `/ecosystem-docs-generate`
update project documentation, then `devops` runs build and tests. **Review and merge:**
`/team-review` runs a parallel multi-dimension code review via agent-teams. `/pr-merge` runs
a final CI check and squash-merges after explicit engineer approval. The issue is closed
after a final confirmation. The entire session is recorded in a structured ailog file
committed to the repository as a permanent audit trail.

## Quick-Start: Common Workflows

### Scope a new feature

```
/conductor:new-track
```

Manual: create a `conductor/tracks/{slug}/spec.md` describing the feature, write a phased
implementation plan in `conductor/tracks/{slug}/plan.md`, then implement task by task.

### Implement a track

```
/conductor:implement
```

Manual: read the track's `plan.md`, implement each task following TDD, commit after each
task, run `npm run lint && npm run format:check && npm test` before opening a PR.

### Build, lint, and format check

```
/dev-build
```

Manual: `npm run format && npm run lint && npm run build`

### Run the test suite

```
/dev-test
```

Manual: `npm test`

### Review a PR

```
/team-review
```

Manual: read all changed files, check against coding standards, open PR comments for each
finding.

### Regenerate ecosystem docs

```
/ecosystem-docs-generate
```

Manual: read `docs/agents/*/design.md`, `.claude/agents/registry.json`,
`.claude/commands/*.md`, then rewrite each file in `docs/claude-ecosystem/` to reflect
current state.
