## Summary

Two small, self-contained fixes: explicitly add `.claude/worktrees/**` to ESLint ignores (#110), and repair the north calibration label boundary bug where `Math.round(5.5) = 6 → % 6 = 0` causes the position one picker-step CCW from north to display 'N' instead of 'NW' (#116).

## Context

The map editor dev tool includes a hex-shaped north-offset picker (`CalibrationControls.vue`) where users click one of 12 positions (0–11) to declare which direction is north on the scanned map image. Six cardinal labels (N, NE, SE, S, SW, NW) rotate around the picker to reflect the selected offset. A rounding boundary bug causes the wrong label at certain offsets.

Separately, the Conductor worktree feature places isolated copies of the repo under `.claude/worktrees/`. Although `.claude/**` is already in the ESLint ignore list, making `.claude/worktrees/**` explicit improves clarity and future-proofs the config if the top-level glob pattern is ever tightened.

## Problem Description

**#110 — ESLint worktree ignore:** `.claude/worktrees/` exists as a live directory; adding it explicitly eliminates any ambiguity and is the canonical recommendation from the Conductor plugin.

**#116 — North label boundary bug:**

- In `cardinalLabelPositions`, `steps = Math.round(((p - n + 12) % 12) / 2) % 6`
- When `(p - n + 12) % 12 = 11` (position is one half-step CCW from north), `11 / 2 = 5.5`, `Math.round(5.5) = 6`, `6 % 6 = 0` → label 'N'
- The correct label is index 5 = 'NW'
- Trigger example: `northOffset = 1` makes the top-edge dot (position 0) display 'N' instead of 'NW'

## Technical Notes

Fix is a one-character swap: `Math.round` → `Math.floor` in `CalibrationControls.vue:67`. This ensures that when a position is exactly halfway between two labels (diff = odd number), it rounds toward the CCW (lower-index) label rather than CW, which is the correct geometric behavior.
