# Specification: HexMapOverlay Performance Quick Wins Bundle

**Track ID:** hex-overlay-perf_20260322
**Type:** Chore
**Created:** 2026-03-22

## Summary

Bundle four deferred performance items in `HexMapOverlay` and its rendering path into a single
low-risk track: rAF throttle on the edge-hover path, deduplication of `findNearestEdge` calls,
a computed property to replace the inline `v-for` filter, and memoization of `wedgePolygonPoints`.

## Context

The map editor renders a dense SVG overlay on every pointer event. PR #88 established the rAF
pattern for hex-hover; PR #158 introduced edge-hover state without applying the same gate.
Four deferred items (#159, #160, #163, #164) cluster in the same file and share a theme of
unnecessary per-frame work. All have clear, low-risk fixes with known patterns in the codebase.

## Acceptance Criteria

- [ ] Edge-hover state writes are gated behind a `requestAnimationFrame` callback, matching the
      hex-hover rAF pattern from PR #88.
- [ ] `findNearestEdge` is called exactly once per mousemove frame (result reused for tooltip
      lookup via `hoverInfo.nearDir`).
- [ ] The inline `.filter()` in the wedge/contour `v-for` is replaced with a computed property.
- [ ] `wedgePolygonPoints` results are memoized so polygon strings are not recomputed between
      calibration changes.
- [ ] All existing Vitest tests pass. Coverage ≥ 70% maintained.
- [ ] `npm run lint` and `npm run format:check` pass.

## Dependencies

- Existing `HexMapOverlay` component and `useEdgePaintTool` composable (PR #158).
- rAF pattern established in PR #88 (reference implementation for the throttle).

## Out of Scope

- `gridData` full-rebuild refactor (#151) — non-trivial geometry split, separate track.

- New rendering features or API changes to overlay props.

## Technical Notes

- `findNearestEdge` dedup (#159): `hoverInfo` already stores `nearDir`; the tooltip lookup
  branch can read from `hoverInfo` rather than calling `findNearestEdge` again.
- rAF gate (#160): mirror the `rafPending` flag pattern from the hex-hover path in
  `HexMapOverlay`; the edge-hover `setEdgeHover` call is the write to gate.
- Computed filter (#163): extract `visibleWedges` (or similar) as a `computed` from the
  inline `.filter()` in the `v-for` directive.
- Wedge memoization (#164): polygon strings only change when `gridSpec` calibration changes;
  a `computed` keyed on `gridData` (which already rebuilds on calibration changes) is sufficient.
