# Specification: OOB Editor Foundation — Launch Script, Store & Hierarchy Tree

## Summary

Wire up the OOB editor dev-tool launch path (npm script, route, home-page nav) and implement the core data layer (`useOobStore.js`) plus the hierarchy tree UI (`OobEditorView`, `OobHierarchyTree`, `OobTreeNode`).

## Context

The map editor and scenario editor dev tools are complete. The OOB editor is the third (and final) dev tool needed before game-logic work begins. Its design is documented in `docs/designs/oob-editor-design.md`. Counter source images and the auto-detection script were landed in the previous track (`counter-detect_20260325`). This track delivers the editor shell and the command-hierarchy tree panel — the foundation that all subsequent OOB editor panels will build on.

## User Story

As a scenario digitizer, I want to launch the OOB editor from the command line and home page, browse the full command hierarchy tree, and have the editor persist my changes automatically, so that I can accurately link counters and stats to units in `oob.json` and `leaders.json`.

## Acceptance Criteria

- [ ] `npm run dev:oob-editor` launches the Vite dev server with the OOB editor route active (mirrors `dev:map-editor`).
- [ ] The home page (`StatusView.vue`) shows a **Dev Tools** section with router-link buttons to both the map editor and OOB editor, conditionally rendered when `VITE_MAP_EDITOR_ENABLED=true`.
- [ ] `useOobStore.js` loads `oob.json` and `leaders.json` via server API → localStorage fallback → bundled JSON import fallback, mirroring the map editor persistence pattern.
- [ ] The store exposes `selectedNode`, `dirty`, and actions for push/pull sync, field updates, and counterRef updates.
- [ ] `OobEditorView.vue` renders a toolbar and two-panel layout (sidebar tree + detail panel placeholder).
- [ ] `OobHierarchyTree.vue` renders the recursive command hierarchy for the active side (Union/Confederate).
- [ ] `OobTreeNode.vue` renders Corps → Division → Brigade → Leader + units with type badges and expand/collapse toggle.
- [ ] All new components have Vitest unit tests; coverage threshold remains ≥ 70%.

## Dependencies

- `counter-detect_20260325` track (landed) — counter source images and `counterRef` schema are in place.
- `docs/designs/oob-editor-design.md` — full OOB editor spec.
- Existing `MAP_EDITOR_ENABLED` env-var pattern in `.env` and `vite.config.js`.
- Existing `scripts/map-editor.sh` as template for `scripts/oob-editor.sh`.

## Out of Scope

- Detail panel (stats editing, leader succession, counterRef linking) — future tracks.
- Server-side OOB API routes — only the existing `/api/data` endpoint is used for initial load.
- Authentication or multiplayer concerns.

## Technical Notes

- Route must be lazy-loaded and guarded by `MAP_EDITOR_ENABLED` (same pattern as `/tools/map-editor`).
- Store persistence mirrors `useMapStore.js`: server fetch → localStorage → static import fallback; debounced auto-save on every edit.
- `OobTreeNode.vue` must be a self-referencing recursive component (use `defineComponent` name or explicit `name` option for Vue to resolve the self-reference).
