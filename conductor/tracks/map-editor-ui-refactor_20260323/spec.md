# Specification: Map Editor UI Refactor — Tool Panel Cleanup & Feature Improvements

**Track ID:** map-editor-ui-refactor_20260323
**Type:** Feature
**Created:** 2026-03-23
**Status:** Draft

## Summary

Refactor all map editor tool panels to remove click/paint mode toggles and streamline editing to click-only; remove bulk elevation ops; restructure the terrain, road, stream, and contour tools with simplified UIs, improved rendering, and better right-click clear behavior.

## Context

The map editor is a Phase 1 dev tool used to digitize the South Mountain game map into `map.json`. The tool panels have accumulated mode-switching complexity (click vs. paint, separate bridge/ford modes) that makes editing cumbersome. The road rendering also needs a visual overhaul: roads should not render for isolated single edges, and should display with outlines for clarity. Bridges and fords need a visual `][` glyph and a more natural workflow as edge types rather than separate modes.

## User Story

As a map editor operator, I want simplified, click-based tool panels with intuitive right-click clearing so that digitizing the game map is fast and unambiguous.

## Acceptance Criteria

- [ ] No Click/Paint mode toggle buttons appear in any tool panel (#180)
- [ ] Elevation tool has no Raise All / Lower All buttons (#181)
- [ ] Terrain tool: no hex icons except buildings; buildings coexist with terrain; right-click clears terrain+building; woodedSloping is visually darker (#182)
- [ ] Road tool: wider outlined roads; no render for 1 road edge; center-to-midpoint lines for 2+ edges; bridge is an edge type button; right-click clears hex roads+bridges; bridge shows black `][` glyph (#183)
- [ ] Stream tool: ford is an edge type button; right-click clears hex streams+walls+fords; ford shows dark blue `][` glyph (#184)
- [ ] Contour tool: no auto-detect button (#185)
- [ ] All CI gates pass (lint, format, test ≥70% coverage)

## Technical Notes

- Building overlay: add `building: boolean` to hex data model; remove `building` from `TerrainType` enum. Migrate existing map data.
- Road rendering: leverage existing through-hex geometry (`edgeToCenter`); add 2+ edge threshold check before rendering; add outline via SVG `<line>` with wider stroke behind the road line.
- Bridge/ford `][` glyph: render as SVG `<text>` or two short `<line>` ticks perpendicular to the edge midpoint, colored black (bridge) or dark blue `#00008b` (ford).
- Right-click on hex (not edge) should clear all features of the relevant group for that hex. Current right-click emits `hex-right-click`; MapEditorView routes this per active tool.
