# Specification: Rendering Architecture — overlayConfig Ownership Cleanup

**Track ID:** overlay-arch-refactor_20260322
**Type:** Refactor
**Created:** 2026-03-22
**Status:** Draft

## Summary

Resolve the three deferred rendering-architecture items (#152, #153, #167) that collectively
violate the tool-owns-its-overlays principle: migrate `HexMapOverlay` to a pure declarative
`overlayConfig` API, move the `overlayConfig` ownership bridge out of `MapEditorView` and
into each tool panel, and collapse the flat 12+ prop list behind the config object.

## Context

PR #145 introduced a declarative `overlayConfig` object alongside the existing explicit props
as a bridge (#152). PR #150 established the tool-panel framework and deferred #153 ("wait until
tool panels exist to own it"). PR #158 delivered edge tool panels (road, stream/wall, contour).
The deferral condition for #153 is now met. Resolving all three together eliminates both parallel
rendering systems and positions `HexMapOverlay` as a clean, config-driven component before Phase 2
game logic is added on top.

## User Story

As a developer adding game-logic overlays in Phase 2, I want `HexMapOverlay` to accept a single
`overlayConfig` object so that I can wire new overlay layers without touching `MapEditorView` or
learning which of 12+ flat props control which behavior.

## Acceptance Criteria

- [ ] `HexMapOverlay` accepts a single `overlayConfig` prop (or structured subset) as its
      primary rendering-control API; legacy flat props that duplicate config keys are removed.
- [ ] Each tool panel (terrain, elevation, road, stream/wall, contour, wedge) owns and
      computes its own `overlayConfig` slice; `MapEditorView` passes them through without
      knowing their contents.
- [ ] The hybrid rendering path (#152) is eliminated: `HexMapOverlay` uses one rendering
      system throughout.
- [ ] No behavior change visible to the user (same map editor UX before and after).
- [ ] All Vitest tests updated to reflect the new prop API. Coverage ≥ 70% maintained.
- [ ] `npm run lint` and `npm run format:check` pass.
