## Summary

Add lightweight enforcement rails to the existing Conductor + Claude workflow to reduce repeated cleanup churn, warning noise, stale debt synchronization risk, and late production-readiness hardening — without changing the default feel of the autonomous process.

## Context

The AI efficiency/code quality assessment (`docs/tech-debt/ai-efficiency-quality-assessment.md`) identified that while the current agentic workflow is effective for fast prototype breadth, it accumulates costs through loose completion criteria and late debt capture. This track extends existing workflow docs and Claude commands so future sessions can implement tracks with clearer done criteria and fewer follow-up debt sprints.

This is documentation-first, intentionally small. No new runtime behavior or application feature changes.
