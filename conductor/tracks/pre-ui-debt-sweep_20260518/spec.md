## Technical Notes

**#361** — `scenario.json` setup entries have no `isDetached` field today. The Garland brigade
group (`_groupNote: "DETACHED"`) and any Confederate detachment entries need `"isDetached": true`
added. `processSetupSide()` must read `entry.isDetached ?? false` and forward it to
`defaultUnit()`. The `UnitStateSchema` refine at line 82 already enforces that
`isDetached: true` requires `orders !== null`.

**#360** — `processReinforcementGroup()` calls `defaultUnit(..., null, false, turn)` ignoring
`group.orderType`. The fix reads `group.orderType ?? null` and passes it as `orderRaw` to
`defaultUnit()`.

**#363** — `GameStateSchema` gains `schemaVersion: z.literal(1).default(1)`. `loadGame()` in
`gameFile.js` calls `GameStateSchema.parse()` which will reject any saved file that set
`schemaVersion` to something else (or missing, because the schema literal enforces value 1).
A stale save produces a Zod validation error with a readable message. No migration utility
needed at this stage.

**#337** — `getScenario()` in `games.js` is a module-level `let _scenario` cache. The fix
adds a `clearScenarioCache()` export (sets `_scenario = null`) and calls it from the scenario
editor's save-success path.

**#364** — Add a `/** null = non-order-holding unit … */ ` block comment above the `orders`
field, reinforcing the existing inline comment. Add an `isOrderHolder(unit)` predicate to
`init.js` (and export it) so query code doesn't need to replicate the `null`-check logic.

**#371** — Refactor the private `defaultUnit(id, hex, orderRaw, isOnBoard, entryTurn, isDetached)`
signature to `defaultUnit({ id, hex, orderRaw, isOnBoard, entryTurn, isDetached = false })`.
Update all five call sites in `init.js`.
