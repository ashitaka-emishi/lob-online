## Phase 2: CounterImageWidget Component

Reusable widget that shows front/back counter thumbnails with upload and filename-edit affordances.

### Tasks

- [x] Task 2.1: Create `client/src/components/CounterImageWidget.vue`
  - Props: `counterRef` (object `{ front, back }` or null), `nodePath` (string)
  - Shows two thumbnail slots (front, back)
  - If filename null or `@error` fires → grey placeholder div + **Browse…** button
  - Browse… opens `<input type="file" accept="image/*">`, POSTs to `/api/counters/upload`, calls `store.updateCounterRef(nodePath, { ...counterRef, [side]: filename })`
  - Editable text input per side; `@change` calls `store.updateCounterRef` without upload
- [x] Task 2.2: Write `client/src/components/CounterImageWidget.test.js`
  - Renders placeholder when counterRef is null
  - Renders `<img>` with correct src when counterRef has filename
  - `@error` on img switches to placeholder
  - Text input change calls store.updateCounterRef

## Phase 3: OobDetailPanel Component

Scrollable form rendering type-specific field sets for the selected node.

### Tasks

- [x] Task 3.1: Create `client/src/components/OobDetailPanel.vue`
  - Props: `node` (object), `nodePath` (string)
  - Computed `nodeType` derived from node structure (regiment/battery/brigade/division/corps)
  - Template sections gated by `v-if` on `nodeType`
  - **Regiment/Battery** fields: id (read-only), name, type select, morale select, weapon/gunType select, strengthPoints number, stragglerBoxes (regiment/cavalry) or ammoClass select (battery), `<CounterImageWidget>`
  - **Brigade** fields: id (read-only), name, morale select, wreckThreshold, wreckTrackTotal, succession placeholder `<p>`, `<CounterImageWidget>`
  - **Division/Corps** fields: id (read-only), name, divisionStragglerBoxes, divisionWreckThreshold, succession placeholder `<p>`, `<CounterImageWidget>`
  - All inputs use `:value` + `@change` → `store.updateField(nodePath + '.fieldName', value)`
- [x] Task 3.2: Write `client/src/components/OobDetailPanel.test.js`
  - Renders correct fields for regiment node
  - Renders correct fields for battery node
  - Renders correct fields for brigade node
  - Renders correct fields for division/corps node
  - Field change calls store.updateField with correct path and value
- [x] Task 3.3: Import and render `<OobDetailPanel>` in `OobEditorView.vue`, passing `store.selectedNode` and its derived path; show "Select a node to edit" placeholder when nothing selected
