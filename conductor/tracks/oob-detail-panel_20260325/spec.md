# Specification: OOB Editor — Detail Panel & Counter Image Widget

## Summary

Implement `OobDetailPanel.vue` — a scrollable form that renders type-specific field sets for regiment, battery, brigade, division, and corps nodes — and `CounterImageWidget.vue` — a reusable thumbnail + upload widget for managing counter image references.

## Context

The OOB editor foundation (hierarchy tree, store, launch script) is complete. The next step is to give the selected-node state meaning: when a user clicks a node in the tree, the detail panel renders the appropriate fields and allows editing. The counter image widget handles the `counterRef` sub-field that appears on every unit type, wiring a file-upload endpoint the server must also expose.

## User Story

As a scenario editor, I want to select any unit in the OOB hierarchy and edit its stats and counter image so that the data files reflect the final digitized scenario.

## Acceptance Criteria

- [ ] Selecting a regiment/battery node shows all correct fields (id, name, type, morale, weapon/gunType, strengthPoints, stragglerBoxes/ammoClass, counterRef widget)
- [ ] Selecting a brigade node shows all correct fields (id, name, morale, wreckThreshold, wreckTrackTotal, successionIds placeholder, counterRef widget)
- [ ] Selecting a division/corps node shows all correct fields (id, name, divisionStragglerBoxes, divisionWreckThreshold, successionIds placeholder, counterRef widget)
- [ ] Counter image widget shows thumbnail when filename resolves, grey placeholder when null or 404
- [ ] Browse… upload flow POSTs to `/api/counters/upload`, updates counterRef with returned filename
- [ ] Editable filename text input updates counterRef without upload
- [ ] All field edits call `store.updateField(path, value)` and set dirty=true
- [ ] `npm run test` passes

## Dependencies

- Server must expose `POST /api/counters/upload` (new endpoint, created in this track)

## Out of Scope

## Technical Notes

- Detail panel receives `nodePath` (dot-path string) and `node` (plain object) as props
- All field selects use `<select>` / `<input>` bound with `:value` + `@change` to call `store.updateField`
- Counter upload endpoint: `multipart/form-data`, saves to `client/public/counters/`, returns `{ filename }`
- Image src: `/counters/{filename}`; 404 handling via `@error` sets a local `imgError` flag
- Use `multer` for file uploads on the server
