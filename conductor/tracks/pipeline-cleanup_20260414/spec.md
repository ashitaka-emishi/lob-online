## Context

The project uses the wshobson/agents conductor + agent-teams plugin as its primary
SDLC workflow. `pr-create` and `pr-merge` predate conductor and may be redundant.
`/tech-debt-report` currently runs as a separate manual step after `/team-review`,
creating friction. The debt register tracks gross scores only, making it hard to
judge whether the project is actually reducing its debt load over time.

## User Story

As the sole developer, I want a simplified, self-contained review-to-debt pipeline
so that debt is always scored at review time and the register reflects true progress.
