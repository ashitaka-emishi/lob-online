# Specification: Click / Paint Modes for Terrain & Elevation Editor

**Track ID:** click-paint-modes_20260318
**Type:** Feature
**Created:** 2026-03-18
**Status:** Draft
**GitHub Issue:** #115

## Summary

Add a click/paint mode toggle to the Terrain Tool and Elevation Tool panels. In click mode (the new explicit default), one click updates one hex. In paint mode, holding the mouse button while moving paints every hex entered continuously.

## Context

The map editor is a dev tool for digitizing the South Mountain terrain data. Rapid bulk editing is a core workflow: the digitizer traces terrain types and elevation contours across 2000+ hexes. The terrain panel already emits `hex-mouseenter` for any hover (not gated on mousedown), which creates accidental paints when hovering while in paint `editorMode`. The elevation panel has no drag mode at all. Both panels need an explicit click/paint toggle with correct mousedown-gating and server-save debouncing.

## User Story

As a map digitizer, I want to toggle between click mode and paint mode in the terrain and elevation panels so that I can quickly paint large areas by dragging without accidentally updating hexes while repositioning my cursor.

## Acceptance Criteria

- [ ] A Click / Paint mode toggle button is visible in both the Terrain Tool panel and the Elevation Tool panel
- [ ] In click mode, terrain and elevation updates fire only on an explicit hex click (no hover painting)
- [ ] In paint mode, `mousedown` on the SVG starts a paint stroke; every hex entered while the button is held receives the current terrain selection or a +1 elevation raise; `mouseup` (or `mouseleave` from the SVG) stops the stroke
- [ ] Paint mode batches `saveMapDraft` calls — at most one draft save fires per stroke (on mouseup), not once per hex
- [ ] The selected mode persists while the panel is open; it does not need to survive page reload

## Dependencies

- `HexMapOverlay.vue` — SVG mousedown/mouseup tracking must be extended beyond `linearFeature` mode
- `useHexInteraction.js` — `onHexMouseenter` gate must respect the new `paintMode`
- `TerrainToolPanel.vue` / `ElevationToolPanel.vue` — UI toggle added to each

## Out of Scope

- Persisting paint mode across page reload
- Paint mode for linear features (already handled via `useLinearFeatureTrace`)
- Right-click drag to lower elevation in paint mode
- Keyboard shortcut for mode toggle (nice-to-have, tracked separately)

## Technical Notes

- `paintMode` is a single shared `ref<'click'|'paint'>` owned by `MapEditorView` — only one tool panel is open at a time, so a single state value covers both panels.
- `HexMapOverlay` already handles `@mousedown` / `@mouseup` / `@mouseleave` for `isDrawing` (linear feature). Extend this pattern: add `isPaintMouseDown` tracking and gate the `hex-mouseenter` emission on `isPaintMouseDown` when `editorMode` is `'paint'` or `'elevation'`.
- Debounce strategy: `saveMapDraft` is currently called inside `onMutated` on every hex update. During a paint stroke, suppress per-hex draft saves and flush once on `mouseup`. Implement via a `paintFlushPending` flag or by passing a `batchMode` option to `useHexInteraction`.
- `ElevationToolPanel` hint text needs updating: click mode = "Click to raise (+1), right-click to lower (−1)"; paint mode = "Drag to raise (+1) continuously".
