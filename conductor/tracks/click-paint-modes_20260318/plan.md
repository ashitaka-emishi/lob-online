## Overview

Three focused changes in dependency order: (1) extend `HexMapOverlay` to track mousedown state and gate `hex-mouseenter` on it, (2) update `useHexInteraction` to respect `paintMode` and debounce draft saves during strokes, (3) add the UI toggle to both tool panels and wire it through `MapEditorView`.

## Phase 1: HexMapOverlay — mousedown gate for paint/elevation modes

Extend the existing SVG mousedown/mouseup handling in `HexMapOverlay` to support non-linearFeature paint modes.

### Tasks

- [x] Task 1.1: Add `isPaintMouseDown` ref (alongside `isDrawing`). In `onSvgMouseDown`, set `isPaintMouseDown = true` when `editorMode` is `'paint'` or `'elevation'`. In `onSvgMouseUp` / `@mouseleave`, always clear `isPaintMouseDown`.
- [x] Task 1.2: Gate the `@mouseenter="emit('hex-mouseenter', cell.id)"` emission: emit only when `editorMode === 'linearFeature' ? isDrawing : isPaintMouseDown` — i.e., keep the emission unconditional for other modes that may need it but add a `v-on` guard so hover-without-mousedown no longer fires `hex-mouseenter` in paint/elevation modes.
- [x] Task 1.3: `defineExpose` — add `isPaintMouseDown` alongside `isDrawing` for test instrumentation.
- [x] Task 1.4: Write / update `HexMapOverlay` tests: verify `hex-mouseenter` is NOT emitted on hover alone when `editorMode='paint'`, IS emitted after `mousedown`, and stops after `mouseup`.

### Verification

- [ ] `npm run test` passes; `HexMapOverlay` tests cover the new gate logic.

## Phase 2: useHexInteraction — paintMode gate + stroke debounce

Update the composable to accept `paintMode` and to debounce draft saves during a stroke.

### Tasks

- [x] Task 2.1: Add `paintMode` param to `useHexInteraction` (default `'click'`). Gate `onHexMouseenter` terrain paint on `paintMode.value === 'paint'` (already gated on `editorMode === 'paint'`; the new condition ANDs with it so hover alone in click mode is fully suppressed).
- [x] Task 2.2: Add elevation paint to `onHexMouseenter`: when `editorMode === 'elevation'` and `paintMode.value === 'paint'`, call `adjustHexElevation(hexId, +1)`. (Right-click to lower stays click-only.)
- [x] Task 2.3: Stroke-scoped draft debounce — accept an optional `onStrokeDone` callback. `MapEditorView` passes a callback that flushes `saveMapDraft` once. Inside `onHexUpdate` (called by `applyPaint` / `adjustHexElevation`), when `paintMode === 'paint'`, skip the immediate `saveMapDraft` call inside `onMutated` and instead set a `strokeDirty` flag. `MapEditorView` fires `onStrokeDone` on `mouseup` to flush.
- [x] Task 2.4: Update `useHexInteraction` tests: click mode suppresses hover paint; elevation paint mode raises on mouseenter; stroke debounce — `saveMapDraft` called once on flush, not per-hex.

### Verification

- [ ] `npm run test` passes; `useHexInteraction` tests cover paintMode gating and debounce.
