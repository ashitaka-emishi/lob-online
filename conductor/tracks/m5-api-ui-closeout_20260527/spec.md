## Context

M5 delivered the turn phase state machine, order system, and basic game map rendering. Three items
remain before M5 is fully closed:

1. The pure `dispatch()` reducer (#355) has no HTTP surface — there is no route that calls it.
2. The GameView component architecture has not been formally designed, blocking safe component splits.
3. The unit info panel is missing counter images, weapon type display, faction header colors, and
   multi-unit paging for stacked hexes.

Issue #387 (playerSide from session) is resolved as part of #356: the action route sources
`playerSide` from `req.session.side`, never from `req.body`.

## Technical Notes

- Socket.io attachment must not break existing routes or require refactoring `server.js` beyond
  minimal wiring; use a separate `socket/` helper module if `server.js` grows.
- The `game:state-updated` event carries only a version token or minimal payload — the client
  fetches authoritative state via REST after receiving it (avoids state-over-socket pitfalls).
- `expectedVersion` in the action body enables optimistic-concurrency: reject if the loaded state
  version does not match, return HTTP 409.
