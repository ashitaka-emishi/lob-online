# Specification: OOB Editor Final UI — Succession List and Leader Detail Panel

**Track ID:** oob-final-ui_20260330
**Type:** Feature
**Created:** 2026-03-30
**Status:** Draft

## Summary

Complete the OOB editor UI by implementing two remaining components: `SuccessionList.vue`
(reusable ordered-list editor for `successionIds` on brigade/division/corps nodes) and the
leader detail panel inside `OobDetailPanel.vue` (all leader fields plus a leader-variant
CounterImageWidget with promoted front/back image slots).

## Context

The OOB editor foundation, detail panel, counter image widget, debt sprint, and server routes
are all shipped. The hierarchy tree renders every node type and selection triggers
`OobDetailPanel.vue`. That component currently handles regiment, battery, brigade, division,
corps, and hq/supply node types — but not leader nodes. `SuccessionList` is referenced in the
design doc but the component file does not exist. Once these two pieces land, the OOB editor is
feature-complete for Milestone M2.

## User Story

As a scenario editor, I want to view and edit leader records and succession lists directly in
the OOB editor so that I can assign counters, update ratings, and set command succession order
without editing JSON by hand.

## Acceptance Criteria

- [ ] Selecting a leader node in the hierarchy tree shows all leader fields (id, name, rank,
      commandLevel, commandsId, initiativeRating, specialRules, counterRef)
- [ ] Leader CounterImageWidget shows front/back rows plus an additional Promoted row
      (promotedFront/promotedBack) with Browse… and filename inputs
- [ ] Browse… on promoted slots uploads via `/api/tools/counters/upload` and updates the store
- [ ] SuccessionList.vue renders the ordered list of leader names resolved from `successionIds`
- [ ] Type-ahead search in SuccessionList filters leaders by side (union/confederate)
- [ ] Add, remove (×), and up/down reorder all update the store via `updateSuccession`
- [ ] Empty succession shows "no succession defined" placeholder
- [ ] Brigade, division, and corps detail panels include a SuccessionList section
- [ ] All field edits set `dirty = true` in the store
- [ ] All new code is covered by Vitest tests (TDD strict)

## Dependencies

- `OobDetailPanel.vue` (extend existing)
- `CounterImageWidget.vue` (extend with `mode="leader"` prop)
- `useOobStore` — `updateSuccession`, `updateCounterRef`, `updateField` (all present)
- `/api/tools/counters/upload` route (shipped in PR #219)
- `successionIds` and `counterRef` fields on command nodes (shipped in PR #219)

## Out of Scope

- Drag-and-drop reordering in SuccessionList (up/down arrows only)
- Pull-confirmation dialog for succession changes (dirty flag is sufficient)
- Army-level node editing (army is display-only in the tree)
- Any game-logic validation of succession order

## Technical Notes

- `CounterImageWidget` currently accepts `nodePath` and `fieldPrefix` props; add `mode` prop
  (`'unit'` default, `'leader'` for promoted row). Keep backward-compatible.
- Leader data lives in `leaders.value` in the store, not `oob.value`. The panel receives the
  node object via prop from the tree — the path prefix (`leaders.union.leaders.0`) identifies
  which store root to use.
- SuccessionList resolves leader IDs to display names by looking up `store.leaders` directly;
  it does not need a separate API call.
- OobDetailPanel already classifies node types via `isRegiment`, `isBrigade`, etc. — add
  `isLeader` computed using `props.nodeType === 'leader'`.
