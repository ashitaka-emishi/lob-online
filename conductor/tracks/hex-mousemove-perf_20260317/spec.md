# Specification: Throttle HexMapOverlay onSvgMouseMove with rAF + Spatial Index

**Track ID:** hex-mousemove-perf_20260317
**Type:** Bug / Performance
**Created:** 2026-03-17
**Status:** Draft
**GitHub Issue:** #88

## Summary

`onSvgMouseMove` in `HexMapOverlay` iterates all grid cells × 6 edge directions (~13,440 distance
calculations) on every mousemove event with no throttling. At 60 Hz this produces ~800,000
calculations/second during map digitization sessions. This fix applies two complementary improvements:
a `requestAnimationFrame` throttle and a spatial index that limits edge checks to the candidate cell
and its 6 neighbors (~42 checks instead of 13,440).

## Context

The map editor is a dev tool used to digitize `docs/reference/sm-map.jpg` into `map.json` terrain
data. Long digitization sessions are the primary use case, making interactive smoothness important.
The issue was flagged during a team-review of PR #86.

## Problem Description

- `onSvgMouseMove` runs the full O(cells × 6) edge-proximity scan on every raw mousemove event.
- At 60 Hz cursor movement this saturates a single CPU core with pure distance math.
- There is also code duplication: `onSvgClick` and `onSvgMouseMove` both contain the edge-nearest
  logic inline.

## Acceptance Criteria

- [ ] `onSvgMouseMove` is throttled with `requestAnimationFrame` (one handler invocation per frame)
- [ ] Edge search uses spatial candidate reduction — only check the cell under cursor + its 6 neighbors
- [ ] `findNearestEdge(localX, localY, cellsToSearch)` helper extracted and shared by both `onSvgClick`
      and `onSvgMouseMove`
- [ ] Edge click and edge hover both continue to work correctly in the map editor
- [ ] `npm run test` passes (green CI)

## Dependencies

- `client/src/components/HexMapOverlay.vue` — primary file to modify
- `honeycomb-grid` `grid.pointToHex` is already available in `onSvgClick`; reuse same pattern

## Out of Scope

- Any changes to edge data structures, `map.json` schema, or server routes
- Worker-thread or WebAssembly offloading
- Performance changes to `onSvgClick` itself (beyond sharing the helper)

## Technical Notes

- Use `allowOutside: false` in `grid.pointToHex` to get a `null` return when cursor is outside the
  grid, and fall back to the full cell list in that case.
- `getCellAndNeighbors` helper should return a de-duped array of cells.
- The rAF flag (`rafPending`) must be component-scoped (ref or plain variable in `<script setup>`).
