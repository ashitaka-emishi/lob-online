# Specification: Elevation System UI — baseElevation + elevationLevels Inputs

**Track ID:** elevation-system-ui_20260318
**Type:** Feature
**Created:** 2026-03-18
**Status:** Draft

## Summary

Add `baseElevation` and `elevationLevels` inputs to the map editor's calibration panel and replace the hardcoded `.max(21)` in `MapSchema` with a runtime check against the stored `elevationLevels` value.

## Context

`elevationSystem.baseElevation` and `elevationSystem.elevationLevels` are top-level fields in `map.json`. `elevationLevels` already has a UI input in `MapEditorView.vue`'s `calibration-extra` section that directly updates `mapData.value.elevationSystem`. `baseElevation` has no UI at all. `HexEntry.elevation` in `map.schema.js` has a hardcoded `.max(21)` (SM-specific), with a comment marking it as pending this issue.

## User Story

As a map digitizer, I want to view and edit both `baseElevation` and `elevationLevels` from the calibration panel so that future scenarios with different elevation ranges can be configured without editing JSON by hand.

## Acceptance Criteria

- [ ] CalibrationControls shows `baseElevation` (int, min 0) and `elevationLevels` (int, min 1, max 99) inputs
- [ ] Changes persist to `map.json` via the existing save/push path
- [ ] Schema `.max(21)` on `HexEntry.elevation` replaced with a `superRefine` check against `elevationSystem.elevationLevels` (defaulting to 22 if absent)
- [ ] Existing SM values (`baseElevation: 500`, `elevationLevels: 22`) pre-populated from `map.json` on load
- [ ] Duplicate `elevationLevels` input removed from `MapEditorView.vue` `calibration-extra` section

## Dependencies

- `map.json` `elevationSystem` structure (already in place)
- `CalibrationControls.vue` and `MapEditorView.vue` (existing files)
- `map.schema.js` `HexEntry` and `MapSchema` (existing)

## Out of Scope

- `contourInterval`, `unit`, `verticalSlopesImpassable` fields (no UI requested)
- `wedgeElevations` min/max hardcoded bounds (separate concern)
- Any game-logic use of `baseElevation`

## Technical Notes

Add `elevationSystem` prop to `CalibrationControls.vue`; emit a new `elevation-system-change` event. In `MapEditorView.vue`, replace the current `calibration-extra` handler with the new event. In `map.schema.js`, remove `.max(21)` from `HexEntry.elevation` and add a `MapSchema.superRefine` that validates per-hex elevation against `elevationSystem.elevationLevels - 1`.
