# Specification: M5 Schema Prerequisites ‚Äî UnitOrderState + isDetached

## Summary

Apply the game-state schema prerequisites needed before the main M5 turn-structure, orders,
socket, and game-map UI work. Introduces `UnitOrderState` (typed order-state object with
`status` and `deliveryTurnDue`), upgrades `UnitStateSchema.orders` from a raw string/null to
`UnitOrderState.nullable()`, adds `UnitStateSchema.isDetached`, and updates the game-state
initializer accordingly.

## Context

M4 delivered `initGameState()`, `GameStateSchema`, and `UnitStateSchema`. The current `orders`
field uses `OrderType` ‚Äî a raw `z.enum(['attack', 'move']).nullable()` ‚Äî which cannot express
"order in the pipeline but not yet delivered" (delay status) or "no order issued" (none
status). This schema change must land before any M5 turn-structure implementation so that
downstream code builds against the correct state shape instead of carrying temporary
string-compatibility shims through the phase engine.

## Problem Description

- `UnitStateSchema.orders` is a raw nullable string. It cannot encode delay status or delivery
  turn ‚Äî both required by the LOB ¬ß10.6 order acceptance/initiative pipeline.
- There is no `isDetached` field; needed for SM detachment rules where a detached brigade
  owns orders independently of its parent division.
- `initGameState()` writes raw string orders (`'move'`, `'attack'`) that will fail validation
  once the schema is upgraded.
