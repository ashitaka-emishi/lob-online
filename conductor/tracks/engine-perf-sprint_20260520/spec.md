## Technical Notes

**#324:** `hexEntryCost` is the single-step public API; Dijkstra callers already hoist the Set
via their `costFn` closure. The fix adds `noEffectSet = null` as a 7th parameter and passes it
through to `hexEntryCostBreakdown`.

**#295:** Early termination fires when the popped hex equals `targetHex`. Since Dijkstra is
correct-by-construction (first pop = optimal cost), breaking immediately after the pop is safe.
`movementRange` passes `null` (no target) and is unaffected.

**#294:** Module-level `Map` caches for both functions. `parseHexId` cache key is the hex ID
string; `formatHexId` cache key is `"${col},${row}"`. Cache is populated lazily. No grid bounds
needed since the map format is globally consistent.

**#201:** `distributeCorpsArtillery` builds `const artyMap = new Map(artyEntries)` once per
corps, then replaces `artyEntries.find(([k]) => k === exactKey)` lookups with
`artyMap.get(exactKey)`. The `endsWith` division-match pattern remains a linear scan (suffix
matching cannot use a Map). Comment updated to reflect the improvement.
