# Specification: Table Test Tool

**Track ID:** table-test-tool_20260412
**Type:** Feature
**Created:** 2026-04-12
**Status:** Draft

## Summary

Build `/tools/table-test` — a standalone dev tool with 11 interactive panels for
exercising every LOB v2.0 game table. Enter parameters and dice rolls; see results
with a full modifier breakdown. No hex map. Backed by existing `engine/tables/*`.

## Context

M3 delivers the full LOB v2.0 rules engine as two testing tools: `/tools/map-test`
(map-based rules — movement, LOS, command range) and `/tools/table-test` (all game
tables — combat, morale, command, charge, leader loss). The map-test tool track is
complete. This is the last M3 deliverable.

All six engine table modules are already implemented and fully tested:
`combat.js`, `morale.js`, `charge.js`, `command.js`, `leader-loss.js`, `weapons.js`.
This track builds the server route layer and client UI on top of them.

## User Story

As a developer verifying LOB v2.0 rules engine output, I want to enter parameters
and dice rolls for any game table so that I can confirm the engine produces correct
results without needing a running game.

## Acceptance Criteria

- [ ] `/tools/table-test` is accessible and guarded by `MAP_EDITOR_ENABLED=true`
- [ ] All 11 panels are reachable from a panel selector on the page
- [ ] Each panel accepts all required inputs defined in the HLD table spec
- [ ] Each panel calls its POST endpoint and renders the result with a modifier breakdown
- [ ] All POST endpoints return 400 on missing or invalid required params
- [ ] `npm run test` passes — all new route and component tests green
- [ ] `npm run lint && npm run format:check` — clean

## Dependencies

- `server/src/engine/tables/combat.js` — `combatResult`, `openingVolleyResult`, column shift helpers
- `server/src/engine/tables/morale.js` — `moraleResult`, `moraleTransition`
- `server/src/engine/tables/charge.js` — `closingRollResult`
- `server/src/engine/tables/command.js` — `commandRollResult`, `orderDeliveryTurns`, `flukeStoppageResult`, `attackRecoveryResult`, `zeroRuleResult`
- `server/src/engine/tables/leader-loss.js` — `leaderLossResult`
- Existing `MAP_EDITOR_ENABLED` guard in `server/src/server.js`
- Existing route pattern from `server/src/routes/mapTest.js`
- Existing Vue panel patterns from `client/src/views/tools/`

## Out of Scope

- No hex map display — this tool is entirely table-driven
- No game-state integration — pure computation, no persistence
- Not wired into the game action/dispatch pipeline (M5/M6 concern)
- No weapon/formation reference browser (weapons.js data is used as input
  metadata only, not displayed as a standalone panel)

## Technical Notes

- Server routes: POST (body params) at `/api/tools/table-test/*`, mounted inside
  the existing `MAP_EDITOR_ENABLED` block in `server.js`
- Client: `TableTestView.vue` with a panel selector; each panel is a self-contained
  SFC in `client/src/components/tools/table-test/`
- No hex interaction — no `HexMapOverlay`, no click-to-select mode
- Modifier breakdown in results should list each modifier by name + value so the
  user can cross-check against the rulebook
