## Technical Notes

**#111 — ElevationSystemControls hoist:** Move the `<ElevationSystemControls>` mount point
from inside `CalibrationControls` to a sibling slot in `MapEditorView`. Remove the
`elevationSystem` prop and `@elevation-system-change` binding from `CalibrationControls`;
wire them directly in `MapEditorView`.

**#112 — Form-input CSS:** Extract shared `label`, `input[type='number']` styles into
`client/src/assets/editor-form.css` and import it in both components. Scoped styles that
are component-specific remain in each `<style scoped>`.

**#154 — ToolChooser validation:** `item.color` is bound to SVG `fill` / CSS `background`.
Add a CSS.supports check (`CSS.supports('color', value)`) in the `itemColor` computed or in
a `validateColor` utility. Emit `console.warn` in development; don't throw.

**#162 — Road/Stream panel dedup:** The two panels share an identical outer chrome: mode
toggle (click/paint), type chooser row, clear-all button, and sub-control slot. Extract
into an `EdgeToolPanelShell` component that accepts the chooser items, sub-control slot,
and emits `mode-change` / `clear-all`.

**#165 — console.assert removal:** `utils/compass.js` uses `console.assert` for dev-only
range checks. Remove them; the assertions add no value in a test-covered codebase and
pollute the hot path.

**#166 — Edge panel CSS dedup:** `RoadToolPanel`, `StreamWallToolPanel`, and
`ContourToolPanel` each have identical `.edge-tool-panel`, `.mode-toggle`, `.mode-btn`,
`.chooser-row`, `.clear-btn` blocks. After #162's `EdgeToolPanelShell` extraction, the
shared CSS moves into the shell component's scoped block; the original blocks are deleted.
(#162 is a prerequisite for #166.)

**#170 — rAF integration test:** Current tests verify that two rapid mousemoves schedule
exactly one rAF callback. The integration gap is that nothing asserts what the callback
actually writes to `hoverInfo`. Add a test that: stubs `requestAnimationFrame` to execute
synchronously, fires a mousemove at a known SVG coordinate, and asserts `hoverInfo`
contains the expected `hexId`, `snapHexId`, `snapDir`, and `inProximity` values.
