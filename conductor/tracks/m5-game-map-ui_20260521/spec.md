# Specification: M5 Game Map UI — Unit Layer, Stats Panel, Game Store

**Track ID:** m5-game-map-ui_20260521
**Type:** Feature
**Created:** 2026-05-21
**Status:** Draft

## Summary

Build the M5 game view: render all units as counter images on the SVG hex map, support
click-to-select with a unit stats panel, and wire a Pinia game store that loads game
state from `GET /api/v1/games/:id` on mount.

## Context

M4 delivered the game API (create/join/list/get), `LobbyView`, and a `GameView.vue`
placeholder stub. The M5 phase engine (`dispatch()`) is complete. `HexMapOverlay.vue`
already handles SVG hex rendering with click events and a declarative `overlayConfig`
prop system. The route `/games/:id` already exists in `client/src/router/index.js`.

The goal for this track is display-only: units visible on the map, selectable, stats
readable. No move/fire/phase controls.

## User Story

As a player, I want to see my units on the hex map and click to select them so that I
can understand the game state and prepare to issue orders.

## Acceptance Criteria

- [ ] `GameView.vue` loads game state from `GET /api/v1/games/:id` on mount, using the
      game ID from the route param (`/games/:id`)
- [ ] All units in the game state are rendered on the hex map as counter images
      (`<image>` elements positioned over hex centers, sized to fit the hex polygon)
- [ ] Clicking a unit's hex selects it; clicking an empty hex deselects
- [ ] A `UnitStatsPanel.vue` renders the selected unit's name, side, strength points,
      morale state, and current order type; shows an empty / "no selection" state when
      nothing is selected
- [ ] `useGameStore.js` (Pinia) holds `gameState`, exposes `loadGame(id)`,
      `selectedUnitId` (ref), and `selectUnit(unitId)` / `deselectUnit()`
- [ ] No movement, fire, or phase UI is included — this track is display-only

## Technical Notes

- `useGameStore.js` lives at `client/src/stores/useGameStore.js`
- `UnitStatsPanel.vue` lives at `client/src/components/UnitStatsPanel.vue`
- `UnitCounterLayer.vue` lives at `client/src/components/UnitCounterLayer.vue` — an
  SVG group that renders `<image>` elements at hex-center coordinates; wired into
  `HexMapOverlay.vue` alongside existing overlay layers
- Counter images are positioned using `hexGeometry.js` center coordinates and sized
  to ~80% of the hex inradius so they fit within the polygon bounds
- `GameView.vue` layout: full-width map area (left) + fixed sidebar (right) holding
  `UnitStatsPanel`; sidebar collapses or shows placeholder when nothing is selected
- The map fetch (`GET /api/tools/map-test/map` or equivalent) already used by
  `MapTestView.vue` provides the `hexes` + `calibration` data needed by `HexMapOverlay`
