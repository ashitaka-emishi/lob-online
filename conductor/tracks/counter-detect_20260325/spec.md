# Specification: Counter Auto-Detection Script

**Track ID:** counter-detect_20260325
**Type:** Feature
**Created:** 2026-03-25
**Status:** Draft
**GitHub Issue:** #197

## Summary

A one-time developer script (`scripts/detect-counters.js`) that uses the Claude vision API to scan
counter images in `docs/reference/src-counters-sm/`, match each to a unit or leader record in
`oob.json`/`leaders.json`, copy **all** images to `client/public/counters/`, write resolved
`counterRef` filenames for high-confidence matches back to both data files, and delete the source
directory once copying is complete.

## Context

The OOB Editor dev tool (M2) requires counter images to be linked to unit and leader records via
`counterRef` fields (`front`/`back` for units; `front`/`back`/`promotedFront`/`promotedBack` for
leaders). The raw source images live in `docs/reference/src-counters-sm/` (filenames like
`CS1-Front_07.jpg`, `CS1-Back_07.jpg`). This script automates the initial linkage so the OOB
Editor has pre-populated data to display and refine.

## User Story

As a developer setting up the OOB Editor, I want to run a single command to auto-detect counter
image assignments so that I do not have to manually match hundreds of counter images to units by
hand.

## Acceptance Criteria

- [ ] Script runs without errors when `src-counters-sm/` contains images
- [ ] All counter images are copied to `client/public/counters/` (original filenames)
- [ ] `docs/reference/src-counters-sm/` is deleted after copying completes
- [ ] High-confidence matches (score ≥ 0.80) are written to data files
- [ ] Low-confidence matches are left as `null` `counterRef` with a note in the report
- [ ] Summary report prints matched/unmatched counts with confidence scores for all images
- [ ] Script is idempotent — running twice does not duplicate entries or re-copy already-copied files
- [ ] `npm run test` still passes (script is not imported by any test)
- [ ] Final version of the script is run to seed OOB editor data and data is committed to the repo

## Dependencies

- `oob.json` — must already have `counterRef` fields added to unit nodes (per OOB Editor data model)
- `leaders.json` — must already have `counterRef` fields added to leader records
- `ANTHROPIC_API_KEY` in environment
- `@anthropic-ai/sdk` npm package (or raw API calls)

## Out of Scope

- Not part of automated build or CI pipeline
- Not triggered by the OOB Editor UI (UI upload is a separate editor feature)
- No TypeScript; script is plain Node.js ES module
- No test coverage required for the script itself (it is a one-shot dev utility)

## Technical Notes

- Confidence threshold: ≥ 0.80 — write to data files; < 0.80 — leave `counterRef` null
- Vision calls: pass image as base64 to Claude (claude-haiku-4-5 or claude-sonnet-4-6 for cost/quality tradeoff)
- counterRef schema per unit: `{ front: filename|null, back: filename|null }`
- counterRef schema per leader: `{ front, back, promotedFront, promotedBack }` all nullable
- Copy strategy: copy ALL images regardless of confidence; only the `counterRef` data write is gated on ≥ 0.80
- Source directory deleted after all images are copied: `docs/reference/src-counters-sm/`
- Idempotency: skip copy if file already exists in `client/public/counters/`; skip source-dir deletion if already gone; skip write if `counterRef` already non-null
- Source directory: `docs/reference/src-counters-sm/`
- Destination directory: `client/public/counters/`
